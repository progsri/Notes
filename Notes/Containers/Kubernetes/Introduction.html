Interacts with kubernetes cluster via API..so it make HTTP calls to master. Self Healing # In Traditional world, if the application is running on 3 servers and one of the server goes down, you would have either reboot that machine or spin another machine and send traffic to it. Either way it is a manual process which cannot be fully automated. In Kubernetes, the 3 instances of the application would reside in 3 different nodes/servers, so if one server or instance goes down, Kubernetes will automatically deploy a new container in another node.
