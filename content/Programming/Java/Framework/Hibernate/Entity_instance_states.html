      merge

      Hibernate identifies a entity based on id, in a session lets an instance was fetched(get()) then the instance is in persistent state, we then update it and flush the instance then it would be in detached state. Know in the same session if the same instance not another instance is updated then it would throw exception, in that case we call merge which puts the instance from detached to persistent state and when we flush it the object would be updated.

      saveOrUpdate

      If the instance is in detached or transient state then save or update call would not work. We then need to use saveOrUpdate.
