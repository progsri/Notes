CRUD operations

All the below createQuery is HQL

t is important to understand from the beginning that all of the methods (persist, save, update, merge, saveOrUpdate) do not immediately result in the corresponding SQL UPDATE or INSERT statements. The actual saving of data to the database occurs on committing the transaction or flushing the Session or closing the session

Select

session.createQuery("from tableName where columnName =''").list(); where tableName and columnName should match with the Entity class.

session.get(Class.class, Obj.getId());

Insert

session.save(obj); This obj is the object of the Entity class

Update

session.createQuery("update tableName set columnName=''").executeUpdate(); where tableName and columnName should match with the Entity class.

session.update(obj); set something on obj before calling the update.

Just retrieve the object using get and call the setter on the object and commit the transaction, no need to call update

Delete

session.createQuery("delete from tableName set columnName=''").executeUpdate(); where tableName and columnName should match with the Entity class.

session.delete(obj) assuming this is a obj which hibernate is aware ( transient ).
