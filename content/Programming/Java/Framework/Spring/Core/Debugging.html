Debugging

Print all the beans created by Spring Container. 


//Autowire ApplicationContext as dependency as 
@Autowired
ApplicationContext applicationContext;

for(String name : applicationContext.getBeanDefinitionNames() ) {
	try{
		System.out.println(name + " -- " + ApplicationContextProvider.getApplicationContext().getBean(name).toString() ) ;
	}catch(Exception ex){
		System.out.println("ERROR " +  name );
	}
}
At runtime to find the property name

After autowiring ApplicationContext to applicationContext doing applicationContext.getEnvironment().getProperty("property-name") gives you the value for the property-name. It can show the properties that the applicationContext is aware of. Remember a application can have more than one ApplicationContext.

applicationContext.getEnvironment().getProperty("property-name") can also read property from System Property , system properties are properties that are passed / configured to JVM ( can also be obtained at runtime by System.getenv() and can also read Environment property , which is operating system properties ( can also be obtained from System.getProperties() )
