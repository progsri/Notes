@SpringBootTest reads the @SpringBootApplication and loads the entire app. Then we can use @AutoconfigureMockMVC which gives MockMVC instance to perform real http calls and see the excepted result, if this http calls has a dependency on service then we can mock that servicevia Mockito and then as usual can verify if the mock service call was done.

Instead of loading the entire app as spring boot as in the case of @SpringBootTest , we can just test the web layer using @WebMVCTest and we can still get access to MockMvc instance to perform http calls.
