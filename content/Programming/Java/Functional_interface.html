java.util.function.Supplier using lambda expression

This is a one of the functional interface, Supplier takes no arguments and always returns a value. To get the return value use get(), this would return value of type T.

The lambda expression such as () -> customMethod() does not take any argument and return a value of type T. So it always return a Supplier.

Call to the method test actually happens when get() is called. 



 static String test(String a){
        return a;
 }
 
  static String test(){
        return new Date().toString();
 }

 Supplier s1 = () -> test("some value");
 s1.get(); // a
 
 Supplier s2 = () -> test();
 s2.get(); // return date in string format

java.util.function.Supplier using method reference operator ( :: )

In the below code ConsumerFunction::test automatically creates the Supplier Function.

:: works for both instance ans static methods. 


public class SupplierFunction {

    static String test(){
        return new Date().toString();
    }

    public static void main(String args[])
    {
       Supplier s1 = SupplierFunction::test;
       s1.get();
    }

}

java.util.function.Consumer

Consumer takes a argument and does not return any value. Type T is the type of argument it takes. To pass an argument use accept method

Call to the method test actually happens when get() is called. 


public class ConsumerFunction {

    static void test(String value){
        System.out.println(new Date().toString());
    }

    public static void main(String args[])
    {
       Consumer c1 = ConsumerFunction::test;
       c1.accept("something"); // Calls the test method at this point, when it takes the argument
    }

}
