Salt

Definition

It is encryption technique which increase the time taken to crack the password.

Generally to store passwords let say in a database, we do not store them as it is, but we generate a hash and store that. So, when the user logins in we regenerate the hash for the user provided password and compare the hash with the hash in our system to authenticate. Know if the database which has these hashes is compromised, then hacker can uses these hashes to find the password strings. Lets see how Salt technique would make it slower to find the passwrod strings.


How to get password from the compromised hashes

Rainbow tables

Hacker maintain a list of all common passwords and their hashes, so if you have a hash and you get the password string, if it is a common password only. This is called a rainbow table which is a very long list.
Which means if the compromised password string is let say abc123. Then the md5sum hash for it would be 2c6c8ab6ba8b9c98a1939450eb4089ed. The chance of having the hash 2c6c8ab6ba8b9c98a1939450eb4089ed in the rainbow table is very high as it is a common one. But if the password is let say "abc123!@#R&*" , which is a rare password, then it would not be in the rainbow table. This is the concept that salt technique uses.
In salt, a random generated string is generated at the time of user registration for every user, this random generated string is called salt, which is then concatenated to the password string and then the hash is generated and stored. So, when ever the user logins in with the password, the authentication process will concatenate the password with the salt and then generate the hash to verify with the hash inthe system. The salt is also stored in plain sight along with the hash.
So, if the hash is compromised the hacker would have to generate all the hashes for the rainbow table with the new salt to find the password. So, If there are a million common passwords in the rainbow table and if the compromised system has 1000 users, then there would be 1000 salts, which means per user, the hacker has to generate new hash value with salt for a entry in the rainbow table.
And if this user does not have common password, this effort is anyway wasted.
