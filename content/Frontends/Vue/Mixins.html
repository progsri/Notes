Mixins

Mixins helps to elimiate code duplication

we can have all the properties for the components (data, computed, methods, filters etc ) in the mixin

Effects from one component does not effect other components...so mixins are locally scoped.

It is possible to have global mixin, however they should be used only for plugins to integrated with third party

main.js 

import Vue from 'vue'
import App from './App.vue'

new Vue({
  el: '#app',
  render: h => h(App)
})
App.vue 

<template>
  <div>
    <sample></sample>
    <sample></sample>
  </div>
</template>

<script>
import Sample from "./Sample.vue";

export default {
  data() {
    return {};
  },
  components: {
    sample: Sample
  }
};
</script>

Sample.vue 

<template>
  <div>
    {{text}}
    <input v-model="text">
    <hr>
  </div>
</template>

<script>
import {genMixin} from "./generalMixin"

export default {
  mixins: [genMixin],
  data() {
    return {};
  }
};
</script>

</style>

generalMixin.js 

export const genMixin = {
    data(){
        return {
            text: "from mixin"
        }
    }
};
