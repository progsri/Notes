Router

This does internal routing instead of going to the server for each request by default, however this behaviour can be changed to go to the server for every request by mode:'history' , otherwise the default would be mode:'hash'.

Router makes our application the SINGLE PAGE APPLICATION

import Vue from "vue";
import VueRouter from "vue-router";
import App from "./App.vue";
import Home from "./views/Home.vue";
import Login from "./views/Login.vue";

Vue.use(VueRouter); //  add vue router module to vue framework

const router = new VueRouter({
  routes: [{ path: "/", component: Home }, { path: "/login", component: Login }]
});

new Vue({
  router,
  render: h => h(App)
}).$mount("#app");

$router

access $router via this

$router has the below 3 functionalities:

  • push
  • push
  • go

The below this.$router.push calls the Login component as it invokes component linked to /login via the router as soon as Home component gets loaded. 



<template>
  <div id="home">Home</div>
</template>

<script>
export default {
  mounted() {
    console.log(
      " Home Component - This is  executed each time the component is mounted or loaded."
    );

    // This would load the login component via the vue router
    this.$router.push("/login");
  },
  data() {
    return {
      msg: ""
    };
  }
};
</script>
