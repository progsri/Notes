The custom JSX element(component) we write can export other custom /in built JSX elements such as h4 span etc.

Using props we can pass data from parent component to the child components as attributes. If props get update to a new value only then react re-renders. It would not re-render if the updated to the existing value.

At class level we can store state locally ( this is not redux which is a global state ). The values of this state can only be accessed from the class ( of course as it is a class field ), if this state is used to render and the state is update to a new value only then react would re-render.It would not re-render if the updated to the existing value.

When possible use stateless components which is just a function or arrow function, when using stateful component i.e class it becomes diffcult to manage.
