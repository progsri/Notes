React Behavior

Based on create-react-app 1_component --scripts-version 1.1.5

The html element are JSX which is nothing but javascript, which means that can be stored in as variable such a let p = (<h2>ABC</h2>),observe the parenthesis. Can be used else with other JSX as {p}

The custom JSX element(component) we write can export other custom /in built JSX elements such as h4 span etc.

Using props we can pass data from parent component to the child components as attributes. If props get update to a new value only then react re-renders. It would not re-render if the updated to the existing value.

At function level component props is local variable, so just use props.xy. At class level component props is a class field, so use as this.props.xy.

using props we can pass data from parent to child. also we can pass parent function from parent to child, which can be invoked by child to send data to parent.

At class level we can store state locally ( this is not redux which is a global state ). The values of this state can only be accessed from the class ( of course as it is a class field ), if this state is used to render and the state is update to a new value only then react would re-render.It would not re-render if the updated to the existing value.

When possible use stateless components which is just a function or arrow function, when using stateful component i.e class it becomes diffcult to manage.

Using props we can pass functions which have access to Parent state from parent component to the child components as attributes. But to access the Parent state we need to bind this properly ... understand this link .

All the css elements in .CSS files are global and these files have to be imported.

There is no specific if else like in vue or angular. As JSX is javascript, we can use for loop or a create a new array with map

Every list should have a key property that is unique for every row in this list, key is used by react to check if there are any chance to the list.

For inline style pass a javascript object.

Use Radium for react styling especially for pseudo selector

Life cycle hooks to create component(only for class based components) in the order below: There are other life cycle hooks but rarely used.

  1. Class constructor(props)
  2. getDerivedStateFromProps(props,state) : Very rarely used
  3. render()
  4. render() of child components
  5. componentDidMount()
Life cycle hooks to update component(only for class based components) in the order below:
  1. getDreivedStateFromProps(props, state)
  2. shouldComponentUpdate(nextProps, nextState) : we can use ths.props or this.state to compare nextProps nd nextState and return true or false based on which the component will update ( next steps in the cycle )
  3. render()
  4. render of child components
  5. getsnapshotBeforeUpdate(prevProps, prevState)
  6. componentDidUpdate()
componentwillUnmount hook is called when the class based component is removed/destroyed/unmounted.

We can use useEffects() as a way to do life cycle hooks for functional components i.e functions which have components. We can have multiple useEffects in the function. useEffects is run each time that component is called or updated. To make it run only for specific variables that the component uses pass second argument as array with the dependent variables. To make it run only when the component is rendered first time pass the second argument as empty array. How to use this when the component is destroyed like componentWillUnmount in case of class based component
