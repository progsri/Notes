Life cycle hooks to create component(only for class based components) in the order below: There are other life cycle hooks but rarely used. Mounting/creating a component

  1. Class constructor(props)
  2. getDerivedStateFromProps(props,state) : Very rarely used
  3. (legacy) componentWillMount() : good place to update state
  4. render()
  5. render() of child components
  6. componentDidMount() :: do not update state as the component is mounted, update state would make react think there are state changes and cause react to render again.
Life cycle hooks to update component(only for class based components) in the order below: Updating a component
  1. getDreivedStateFromProps(props, state)
  2. shouldComponentUpdate(nextProps, nextState) : we can use ths.props or this.state to compare nextProps nd nextState and return true or false based on which the component will update ( next steps in the cycle )
  3. render()
  4. render of child components
  5. getsnapshotBeforeUpdate(prevProps, prevState)
  6. componentDidUpdate()
componentwillUnmount hook is called when the class based component is removed/destroyed/unmounted Unmounting a component .

We can use useEffects() as a way to do life cycle hooks for functional components i.e functions which have components. We can have multiple useEffects in the function. useEffects is run each time that component is called or updated. To make it run only for specific variables that the component uses pass second argument as array with the dependent variables. To make it run only when the component is rendered first time pass the second argument as empty array.
