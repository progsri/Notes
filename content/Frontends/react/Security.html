Security

Authorization from backend

  • When the user enters the login information when clicked on sign in modal
  • Show the spinner
  • Make a Promise call
  • if this is success then add the token and expiration in the localStorage via the redux middleware function and enable the authentication in the redux via dispatching a action to the reducer, this is the flag rest of the app depends on for authentication check and then set a timeout settimeout for remaining time in expiration and then Redirect or props.history to Home page.
  • if failure show error in the sign in page.

Authentication for subsequent requests

  • Let's say the user opens the app
  • use localStorage to retreive the token and expiration from the browser
  • if expired then use Redirect or props.history to redirect to sign in page.
  • if not expired then mark the authorized flag in redux as true and set a timeout settimeout for remaining time in expiration and then Redirect or props.history to Home page.

XSS Cross site scripting

This is the technique in which hacker can get the cookie and other information of the user's browser by embedding javascript code that gets executed when the user clicks on a link asssuming it is a genuine one.

React by default provides this protection. may be it removes the < and > symbols
