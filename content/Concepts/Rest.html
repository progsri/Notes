Best practices

Send back appropriate HTTP codes.

  • 200 : all success
  • 400 : If validation fails
  • 500 : If backend has issues, lets say exception

Always send a response, both sucess and failure scenarios. 

 
  {
    data:{} //has data if success
    hasErrors: true or false 
    errors: [] // list of errors
    timestamp: timestamp
  }

Document the service using swagger or similar tools, make sure that api can called to interact with the service.

Use appropritate HTTP method.

  • GET : to retrieve a resource
  • POST : Insert resource
  • PUT : Update resource
  • DELETE : delete a resource
  • PATCH
  • HEAD
