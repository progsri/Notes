Code Review

Look if @Deprecated is placed, if it is then the comment should mention what is the new method/class

No long calls when @Transactional is used. do not make http calls when transaction is under the opened in the RDBMS , under large volume traffic these cause deadlocks.

Do write continous large logic, break it into small chunks.

when possible to boost the response time use threads or executor service.

Use pools, connection pools for database or thread pools.

In Java, do not return null.

Structure of the application, if API then controller → service → dao. Keep utilites/helper classes seperate.

Please be detailed in Git commits, will be every helpful later.

Documentation for API, you can use swagger.

If data does not change much, please do cache it.

Use annotations and AOP for security, transactions, threading, caching. By having the logic inside the annotations and by using the AOP concepts we can have the method for business logic only.

Method and variable names should be explicity even if they are long.

If using hibernate, use Lazy loading otherwise there will be a performance hit if OneToMany or ManyToMany.
