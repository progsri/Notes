minikube setup for runnning kubernetes locally

Install kvm2 driver 


Download https://storage.googleapis.com/minikube/releases/latest/docker-machine-driver-kvm2 
chmod +x docker-machine-driver-kvm2 
sudo cp docker-machine-driver-kvm2 /usr/local/bin/ 
chmod +x /usr/local/bin/docker-machine-driver-kvm2
rm docker-machine-driver-kvm2

Install minikube 


Download https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
cp minikube-linux-amd64 minikube
sudo install minikube /usr/local/bin

Start minikube

minikube start --vm-driver kvm2 . if any issue rm -rf ~/.minikube

Minikube's docker

minikube comes with it's own docker, it does not use your local machine docker daemon. So to see the images or containers inside the minikube docker you need to point to it by running eval $(minikube docker-env) , this simply exports the below variables to your present terminal, this is temporary and has to be run in all the terminals. 


⇒  minikube docker-env                                     
export DOCKER_TLS_VERIFY="1"
export DOCKER_HOST="tcp://192.168.39.32:2376"
export DOCKER_CERT_PATH="/home/progsri/.minikube/certs"
# Run this command to configure your shell:
# eval $(minikube docker-env)
