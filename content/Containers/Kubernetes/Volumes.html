Volumes

A pod stores data internally, so if a pod is terminated or destroyed we will loose data. if a new pod is created it would be able to read that data but how can it do that if the pod is created in a completely different node.

To persist data we can use either Persistent volume or Persistent volume claims

Persisten Volumes

Data will be stored outside of the pod rather than internal to the pod.

What if the node on which the data resides dies ?

Persistent Volume claims

Not a real volume.

Indicates list of possible volumes.

Access modes:

  • ReadWriteOnce : read and write on single node
  • ReadOnlyMany : read on multiple nodes
  • ReadWriteMany : read and write on multiple nodes
